# 第1段階：ローカルで試作版を動かす手順

この段階の目的は、ログインなし・本文表示なしで、設問、入力欄、添削ボタンが動くことを確認することです。

## 0. まだやらないこと

この段階では、次のものはまだ入れなくて大丈夫です。

- Teamsログイン
- 生徒ごとの履歴保存
- 公開期間制限
- 本番公開

まずは先生のPCで動けばOKです。

## 1. 必要なもの

- Node.js
- VS Code
- このフォルダ一式
- OpenAI APIキー（なくても簡易添削モードで動作確認できます）

## 2. フォルダを開く

VS Codeで、この `genbun-tensaku-prototype` フォルダを開きます。

## 3. ターミナルで準備

VS Codeのターミナルで次を実行します。

```bash
npm install
```

## 4. APIキーなしで動作確認

次を実行します。

```bash
npm run dev
```

表示されたURLをブラウザで開きます。

だいたい次のようなURLです。

```text
http://localhost:3000
```

この状態ではOpenAI APIキーがないため、AI添削ではなく「簡易添削モード」で動きます。
画面確認にはこれで十分です。

## 5. AI添削に切り替える場合

`.env.example` をコピーして、同じ場所に `.env.local` を作ります。

中身を次のようにします。

```env
OPENAI_API_KEY=ここにOpenAIのAPIキーを貼る
```

例：

```env
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

その後、いったんターミナルを止めます。

```bash
Ctrl + C
```

もう一度起動します。

```bash
npm run dev
```

これでAI添削モードになります。

## 6. この試作版の仕様

生徒画面に表示されるもの：

- 作品名
- 設問
- 答案入力欄
- 添削ボタン
- 添削結果

生徒画面に表示されないもの：

- 本文全文
- 模範解答
- 採点基準

模範解答や採点基準は、サーバー側の `lib/questionBank.ts` に入っています。
